SECURITY INCIDENT INVESTIGATION REPORT

Incident Type: Phishing → Credential Theft → Suspicious Foreign Login
Severity: High Affected System: Microsoft 365 / Corporate Email Target
User: j.smith@acme-corp.com Detection Source: SIEM alert (Impossible
Travel / Suspicious Login)

1.  EXECUTIVE SUMMARY A phishing email impersonating Microsoft 365
    Security was delivered to a finance department employee. The email
    contained a malicious link that redirected the user to a fake
    Microsoft login page designed to steal credentials.

After the victim entered credentials on the phishing page, the attacker
used the stolen credentials to authenticate from a foreign IP address
located in Russia. The attacker successfully accessed the victim’s
mailbox and created a malicious email forwarding rule to exfiltrate
sensitive communications.

The activity triggered a SIEM alert for suspicious login behavior,
indicating a potential account compromise.

2.  SUSPICIOUS ACTIVITY IDENTIFIED

3.  ATTACKER BEHAVIOR ANALYSIS

pretending to be a Microsoft security alert.

Step 2 — Credential Harvesting The victim clicked the phishing link and

This rule allows continuous monitoring of incoming emails.

9.  ROOT CAUSE

The compromise occurred because the victim entered credentials into a
phishing website that impersonated Microsoft 365 authentication
services. The attacker used the stolen credentials to access the account
from a foreign IP address. Multi-factor authentication was not enforced
or was bypassed.

10. RECOMMENDED RESPONSE ACTIONS

Immediate Containment • Reset the compromised user password • Revoke all
active authentication sessions • Remove malicious mailbox forwarding
rules • Block malicious IP addresses • Block phishing domains at the
email gateway and DNS layer

Detection Improvements • Implement SIEM alerts for impossible travel
logins • Monitor mailbox rule creation events • Detect logins from
high-risk geographic locations • Monitor SPF/DKIM/DMARC authentication
failures

Security Hardening • Enforce MFA across all accounts • Implement
phishing-resistant authentication methods • Deploy email security
filtering and sandboxing • Conduct employee phishing awareness training

11. FINAL ASSESSMENT

This incident represents a successful phishing-based account takeover.
The attacker leveraged stolen credentials to access the victim’s mailbox
and attempted persistence using email forwarding rules for data
exfiltration. Rapid detection by the SIEM allowed the security team to
investigate and contain the incident.
